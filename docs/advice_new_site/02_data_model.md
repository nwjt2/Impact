# 02 — Data Model

CSV-first. Source-of-truth data lives in plain CSVs; SQLite is a derived view rebuilt from CSVs by the webapp's import step. This is more durable than a DB-first approach for a solo operator: you can hand-edit, diff in git, and recover from anything.

## Identifiers

Compound keys with slug-style names. Examples:
- Fund unique: `<INGO_Slug>_<Fund_Slug>`
- LP-Fund edge: `<LP_Slug>__<Fund_Slug>`
- Investor-Company edge: `<Investor_Slug>__<Company_Slug>__<Round>`

Slugs use lowercase + hyphens; never spaces; strip "Inc", "Ltd", "Capital", "Partners" suffixes consistently. Centralize the slugify function in `utils/` and use it everywhere — do not let scrapers each invent their own.

## Master registries (generated, never hand-edited)

- `master_id_registry.csv` — assigns and persists UIDs across runs so identifiers don't churn.
- `master_slug_registry.csv` — generated by `build_slug_registry.py` from the name registry. **Never hand-edit.** Manual entries cause UID/slug mismatches that bite weeks later.

## Operator override CSVs (hand-edited, version-controlled)

- `manual_matches.csv` — when fuzzy name-matching is wrong (e.g. "Acme Capital" vs "Acme Capital Partners"), operator pins the canonical mapping.
- `manual_field_resolutions.csv` — operator pick when two scrape sources disagree on a field.
- `discovery_skip_list.csv` — entities that can't be scraped, with a reason. Prevents re-trying broken sources every run.
- `dismissed_warm_leads.csv` — investors the operator has researched and ruled out, so they stop appearing in the lead view.

## CSV column conventions

- Title case with spaces in headers: `"Fund Name"`, `"Investor Slug"`.
- UTF-8, no BOM.
- Quoted fields when they contain commas or newlines.
- Booleans as `Y` / `N` / `unknown` (not `true`/`false`/`null`).
- Dates: `YYYY-MM-DD` or `YYYY-MM` if day unknown.
- Pipe-separated lists for multi-value columns: `"health|africa|early-stage"`.

## A note on data sources

Be realistic about what's scrapeable. **LP rosters are the hardest** — funds rarely publish them. Sources to explore in priority order:

1. Fund's own "Our Backers" / "Investors" / "Supported by" page.
2. INGO's annual report PDFs (parse with pdfplumber).
3. Press releases announcing fund launches and closes.
4. SEC Form D filings (US funds).
5. Crunchbase / PitchBook (paid, last resort).

For portfolio companies: their own "About / Investors" sections, then press releases, then Crunchbase. Build the easy ones first; do not chase 100% coverage on hard sources before the easy network is wired up end-to-end.
